## 1. 시스템 아키텍처

### 1.1 전체 구성

```
┌─────────────────────────────────────────────┐
│              모바일 웹 브라우저                │
│        (Vue 3 + TypeScript / PWA)            │
└───────────────────┬─────────────────────────┘
                    │ HTTPS / REST
                    ▼
┌─────────────────────────────────────────────┐
│            FastAPI (Python)                   │
│            배포: Render                       │
│  ┌──────────┬──────────┬──────────────────┐  │
│  │ 자산 API  │ 가족 API  │ FDS / 유산 API    │  │
│  └──────────┴──────────┴──────────────────┘  │
└──────┬──────────────┬───────────────┬────────┘
       │              │               │
       ▼              ▼               ▼
┌────────────┐ ┌────────────┐ ┌────────────────┐
│  Supabase  │ │ LLM (챗봇) │ │ OCR / STT      │
│ PostgreSQL │ │  Gemini    │ │ Tesseract/Web  │
│ + Auth     │ │  (무료)    │ │ Speech (무료)  │
└────────────┘ └────────────┘ └────────────────┘
```

### 1.2 기술 스택

| 레이어 | 기술 | 배포/호스팅 | URL |
|--------|------|-------------|-----|
| 프론트엔드 | Vue 3 + TypeScript | Vercel | coverletter-ai-frontend.vercel.app |
| 백엔드 | FastAPI + Python | Render | coverletter-ai-backend.onrender.com |
| DB | PostgreSQL | Supabase | — |
| 인증 | Supabase Auth | Supabase | — |

> 배포 URL은 현재 `coverletter-ai`로 되어 있어, 이음 프로젝트 확정 시 도메인 정리 권장.

### 1.3 AI / 외부 서비스 (MVP 무료 구성)

| 기능 | MVP 기본 (무료) | 유료 업그레이드 옵션 | 비고 |
|------|----------------|---------------------|------|
| 챗봇 / LLM | **Google Gemini API** (무료 티어) | OpenAI GPT-4o | 분당 15회 · 일 1,500회 무료 |
| 챗봇 대안 | Groq (Llama 3.3 70B) | — | 속도 빠름, 무료 |
| OCR | **Tesseract.js** (브라우저) | CLOVA OCR / Google Vision | 서버 불필요, 한글 정확도 낮음 |
| 음성 STT | **Web Speech API** (브라우저) | Google Speech-to-Text | iOS Safari 부분 지원 주의 |

> MVP 기본 구성으로는 **운영 비용 0원** 달성 가능. 정확도 이슈 발생 시 OCR만 Google Vision(월 1,000건 무료)로 교체 권장.

## 2. 데이터베이스 설계 (Supabase / PostgreSQL)

### 2.1 ERD 개요

```
users 1 ──── N assets
  │
  ├── 1 ──── N helpers (자기 참조: senior ↔ helper)
  │
  ├── 1 ──── N digital_legacy
  │
  └── 1 ──── N fds_logs
```

### 2.2 테이블 정의

#### `users` — 사용자

| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| id | uuid | PK, default uuid | Supabase Auth 연동 |
| email | text | unique, not null | 로그인 식별자 |
| name | text | not null | 표시 이름 |
| role | text | not null | `senior` / `helper` |
| created_at | timestamptz | default now() | 가입 시각 |

#### `assets` — 자산

| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| id | uuid | PK | |
| user_id | uuid | FK → users.id | 소유 시니어 |
| asset_type | text | not null | `bank` / `insurance` / `realestate` / `crypto` 등 |
| asset_name | text | not null | 자산명 (예: 농협 예금) |
| institution | text | nullable | 금융사명 |
| amount | bigint | nullable | 금액(원), 미상 시 null |
| source | text | not null | 입력 방식: `voice` / `ocr` / `manual` |
| created_at | timestamptz | default now() | |
| updated_at | timestamptz | default now() | |

#### `helpers` — 조력자 연결

| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| id | uuid | PK | |
| senior_id | uuid | FK → users.id | 시니어 |
| helper_id | uuid | FK → users.id | 조력자 자녀 |
| permission_level | text | not null | `read_only` (MVP 고정) |
| invited_at | timestamptz | default now() | 초대 시각 |
| accepted_at | timestamptz | nullable | 수락 시각 |
| status | text | not null | `pending` / `active` / `revoked` |

#### `digital_legacy` — 디지털 유산 노트

| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| id | uuid | PK | |
| user_id | uuid | FK → users.id | 소유자 |
| category | text | not null | `sns` / `crypto` / `subscription` |
| title | text | not null | 항목명 (예: 카카오 계정) |
| encrypted_data | text | not null | 암호화 저장 (AES) |
| trigger_type | text | not null | `date` / `manual` |
| trigger_date | date | nullable | 날짜 기반 트리거 시 |
| created_at | timestamptz | default now() | |

#### `fds_logs` — 이상 감지 로그

| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| id | uuid | PK | |
| user_id | uuid | FK → users.id | 대상 시니어 |
| pattern_type | text | not null | `time` / `amount` / `device` |
| detail | jsonb | nullable | 감지 상세 |
| is_locked | boolean | default false | 긴급 잠금 여부 |
| detected_at | timestamptz | default now() | 감지 시각 |

### 2.3 Row Level Security (RLS) 정책 요약

| 테이블 | 정책 |
|--------|------|
| assets | 시니어 본인 = 전체 CRUD / 연결된 조력자 = SELECT만 |
| helpers | senior_id 또는 helper_id 일치 시 조회 |
| digital_legacy | 본인만 접근 |
| fds_logs | 본인 + 연결 조력자 SELECT |

> 조력자의 **이체·수정 차단**은 RLS에서 SELECT 권한만 부여하여 구조적으로 보장.

## 3. API 명세 (FastAPI)

### 3.1 공통 규약

- **Base URL**: `https://coverletter-ai-backend.onrender.com/api/v1`
- **인증**: `Authorization: Bearer <supabase_jwt>`
- **응답 형식**: JSON, snake_case
- **공통 응답 래퍼**:

```json
{
  "success": true,
  "data": { },
  "error": null
}
```

### 3.2 엔드포인트 목록

#### 인증

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/auth/login` | Supabase 소셜 로그인 콜백 처리 |
| GET | `/auth/me` | 현재 사용자 정보 |

#### 자산

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/assets` | 자산 목록 조회 |
| POST | `/assets` | 자산 직접 추가 |
| POST | `/assets/voice` | 음성 텍스트 → 자산 파싱 |
| POST | `/assets/ocr` | 이미지 → OCR 자산 추출 |
| PATCH | `/assets/{id}` | 자산 수정 |
| DELETE | `/assets/{id}` | 자산 삭제 |

#### 가족 / 조력자

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/helpers/invite` | 조력자 초대 (이메일) |
| POST | `/helpers/accept` | 초대 수락 |
| GET | `/helpers` | 연결된 조력자 목록 |
| GET | `/helpers/dashboard/{senior_id}` | 조력자용 자산 대시보드 (read-only) |

#### FDS / 긴급 잠금

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/fds/lock` | 긴급 잠금 실행 |
| POST | `/fds/unlock` | OTP 인증 후 해제 |
| GET | `/fds/logs` | 이상 감지 로그 조회 |

#### 디지털 유산

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/legacy` | 유산 노트 목록 |
| POST | `/legacy` | 유산 항목 추가 (암호화 저장) |
| DELETE | `/legacy/{id}` | 항목 삭제 |

### 3.3 주요 요청/응답 예시

#### POST `/assets/voice`

음성 인식 결과 텍스트를 LLM으로 파싱해 자산 후보 반환.

요청:
```json
{ "text": "농협에 돈 넣어뒀어" }
```

응답:
```json
{
  "success": true,
  "data": {
    "candidates": [
      {
        "asset_type": "bank",
        "institution": "농협은행",
        "asset_name": "예금",
        "amount": null,
        "confidence": 0.82
      }
    ]
  },
  "error": null
}
```

> 서버는 Gemini API에 구조화 프롬프트를 전달하고, JSON만 반환하도록 강제 후 파싱.

#### POST `/assets/ocr`

요청: `multipart/form-data` (이미지 파일)

응답:
```json
{
  "success": true,
  "data": {
    "extracted": {
      "institution": "국민은행",
      "asset_name": "정기예금",
      "amount": 5000000
    },
    "raw_text": "국민은행 정기예금 ..."
  },
  "error": null
}
```

> OCR은 클라이언트(Tesseract.js)에서 1차 처리 후 텍스트만 서버로 보내는 방식도 가능 → 서버 부하·비용 절감.

## 4. 프론트엔드 구조 (Vue 3 + TypeScript)

### 4.1 디렉터리 구조

```
src/
├── assets/              # 정적 리소스
├── components/          # 재사용 컴포넌트
│   ├── common/          # 버튼, 카드, 모달 등
│   ├── senior/          # 시니어 모드 전용
│   └── helper/          # 조력자 모드 전용
├── views/               # 페이지 단위
│   ├── HomeView.vue
│   ├── AssetListView.vue
│   ├── VoiceInputView.vue
│   ├── OcrInputView.vue
│   ├── HelperDashboardView.vue
│   └── LegacyNoteView.vue
├── composables/         # 로직 재사용 (useAuth, useAssets ...)
├── stores/              # Pinia 상태 관리
├── services/            # API 호출 계층 (axios)
├── types/               # TypeScript 타입 정의
├── router/              # Vue Router
└── App.vue
```

### 4.2 상태 관리 (Pinia)

| Store | 역할 |
|-------|------|
| `authStore` | 로그인 상태, 사용자 role |
| `assetStore` | 자산 목록 캐시, CRUD |
| `helperStore` | 조력자 연결, 대시보드 데이터 |
| `fdsStore` | 잠금 상태, 알림 |

### 4.3 모바일 우선 UI 원칙

| 항목 | 기준 |
|------|------|
| 반응형 | Tailwind CSS, mobile-first 브레이크포인트 |
| 시니어 글자 크기 | 본문 최소 18px, 제목 24px+ |
| 터치 영역 | 버튼 최소 44×44px |
| 카메라 호출 | `<input type="file" accept="image/*" capture="environment">` |
| 음성 입력 | Web Speech API, 미지원 브라우저는 텍스트 입력 fallback |
| PWA | manifest + service worker (선택, 앱 설치형 경험) |

### 4.4 API 연동 계층 예시

```typescript
// services/assetService.ts
import { api } from './client'
import type { Asset, VoiceParseResult } from '@/types'

export const assetService = {
  list: () =>
    api.get<{ data: Asset[] }>('/assets'),

  parseVoice: (text: string) =>
    api.post<{ data: VoiceParseResult }>('/assets/voice', { text }),

  create: (payload: Partial<Asset>) =>
    api.post<{ data: Asset }>('/assets', payload),
}
```

```typescript
// types/asset.ts — API 응답과 1:1 매핑
export interface Asset {
  id: string
  user_id: string
  asset_type: 'bank' | 'insurance' | 'realestate' | 'crypto'
  asset_name: string
  institution: string | null
  amount: number | null
  source: 'voice' | 'ocr' | 'manual'
}
```

## 5. 개발 순서 (권장)

```
1. Supabase DB 스키마 생성 + RLS 정책
        ↓
2. FastAPI API 스펙 정의 (응답 형식 확정)
        ↓
3. [병렬] 디자인 시스템 (색상·타이포·컴포넌트 규칙)
   [병렬] 프론트 컴포넌트 구조 + mock 데이터 개발
        ↓
4. 백엔드 실제 구현 (AI 연동 포함)
        ↓
5. 프론트 ↔ 백엔드 실제 API 연동
        ↓
6. 모바일 기기 테스트 (iOS STT 확인 필수)
```

> 핵심: **DB 스키마 → API 스펙**이 먼저 확정되어야 프론트가 mock으로 병렬 개발 가능.
